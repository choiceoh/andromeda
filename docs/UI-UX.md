# Andromeda UI·UX 디자인 문서

> **"Warm Zen" 디자인 시스템** — Andromeda 데스크탑 워크스테이션의 시각 언어·
> 상호작용·컴포넌트 규약의 단일 진실원. [`DESIGN.md`](DESIGN.md)가 _왜·무엇을_(아키텍처
> ·정체성)이라면, 이 문서는 *어떻게 보이고 작동하는가*다. 새 화면을 만들기 전에 읽고,
> 구조를 바꾸면 갱신할 것.

구현 진실원: `src/styles.css`(`:root` 토큰·클래스) + `src/theme.ts`(인라인 미러).
이 문서는 그 코드를 설명하지, 대체하지 않는다. 값이 어긋나면 코드가 옳다.

---

## 1. 디자인 철학

> 이 절은 시스템에 필요한 만큼의 _요약_ 이다. 신념의 확장판과 설계 판단법은
> [`DESIGN-PHILOSOPHY.md`](DESIGN-PHILOSOPHY.md).

Andromeda는 데네브 게이트웨이의 **데스크탑 사령탑**이다. 메일·일정·할일·위키를 밀도
높게 다루면서, 그 옆에서 AI가 "지금 뭐가 중요한가"를 총괄 보조한다. 도구가 아니라
**작업 환경**이므로, UI는 콘텐츠를 압도하지 않고 종일 켜둬도 피로하지 않아야 한다.

미감은 **"Warm Zen"** — Zen 브라우저에서 빌려온 _떠 있는 둥근 패널_ 은유에 따뜻한
점토(clay) 톤 하나를 더했다.

| 키워드            | 의미                                                                      |
| ----------------- | ------------------------------------------------------------------------- |
| 떠 있음(floating) | 패널이 부드러운 그라디언트 위에 그림자로 떠 있다. 꽉 찬 박스 UI가 아니다. |
| 따뜻함(warm)      | 차가운 중성 배경에 따뜻한 글로우. 단일 액센트는 점토색(`#c17a5b`).        |
| 고요함(zen)       | 절제된 모션, 얇은 헤어라인, 조용한 스크롤바. 장식 없음.                   |
| 밀도(dense)       | 업무 데이터는 한눈에 — 촘촘한 그리드, 작은 행간, 정보 우선.               |

### 핵심 원칙 (절대 준수)

1. **하드코딩 색상 금지.** 컴포넌트는 `:root` 토큰(`var(--…)`) 또는 `styles.css`
   클래스만 참조한다. 인라인은 `theme.ts`의 토큰 미러를 쓴다. 생 hex는 `styles.css`의
   `:root` 정의에만 존재한다.
2. **액센트는 하나.** 점토색(`--accent`)이 유일한 포인트 컬러다. 의미 색(온라인·위험)을
   빼면 두 번째 색을 도입하지 않는다.
3. **AI는 픽셀이 아닌 텍스트를 본다.** 모든 패널은 내용을 텍스트로 직렬화해
   `useRegisterPane`로 AI 패널에 밀어 넣는다. 화면에 보이는 것 == AI가 읽는 것.
4. **모션은 거든다, 끌지 않는다.** 진입 페이드·상태 펄스·150ms 전이까지. 그라디언트·
   그림자 남용·튀는 애니메이션 금지. `prefers-reduced-motion` 존중.
5. **밀도 우선, 그러나 12px 미만 금지.** 데이터는 촘촘히, 그러나 최소 가독 크기를 지킨다.
6. **연결 안 됨이 기본 상태.** 게이트웨이 없이도 모든 화면이 깨지지 않고 안내를 보여준다
   (CI·샌드박스엔 라이브 게이트웨이가 없다).

## 2. 레이아웃 · 정보 구조

워크스테이션은 윈도우 그라디언트 위를 떠다니는 **3열 셸**이다. 셸 자체는 투명하고,
패널만 떠 있다. (`Workstation.tsx`)

```
┌─ 윈도우: --grad 그라디언트 (고정) ──────────────────────────────────┐
│  ▌레일      ┌──────── 작업 영역 (.panel) ───────┐  ┌─ Deneb AI ─┐  │
│  100px      │  활성 패널 1개만 렌더              │  │  352px     │  │
│  ▌오늘      │  h2 제목 + 그리드/에디터/폼        │  │ ─────────  │  │
│  ▌할일      │                                    │  │ 능동 알림  │  │
│  ▌문서      │  PeoplePane · MailPane · …         │  │ 컨텍스트   │  │
│  ▌메일      │                                    │  │ 스트림 응답│  │
│  ▌ …        │                                    │  │ ─────────  │  │
│  · 연결점   └────────────────────────────────────┘  │ 입력 · ↑   │  │
└──────────────────────────── gap: 12px ──────────────────────────────┘
```

| 영역          | 폭                 | 역할                                                | 컴포넌트            |
| ------------- | ------------------ | --------------------------------------------------- | ------------------- |
| 네비 레일     | `--rail-w` (100px) | 패널 전환 탭(아이콘+라벨) + 하단 연결 상태점/팝오버 | `Sidebar`           |
| 작업 영역     | `flex: 1` (가변)   | 활성 패널 1개만 렌더. 그리드·에디터·대시보드        | `main.panel` + 패널 |
| Deneb AI 패널 | `--ai-w` (352px)   | 능동 알림 → 컨텍스트 안내 → 스트림 응답 → 입력      | `AIPanel`           |

- **셸 패딩·간격:** 전체에 `--gap`(12px) 패딩, 열 사이 `--gap` 간격. 작업 패널 내부는
  `20px 22px`, AI 패널은 `16px`.
- **단일 패널 렌더:** 작업 영역은 활성 패널 하나만 그린다. 탭·라우팅·단축키는 모두
  패널 레지스트리(`panes/index.ts`)에서 파생된다.
- **반응형:** 레일·AI 패널은 고정폭, 작업 영역만 신축. 데스크탑 단일 윈도우 가정
  (멀티윈도우는 Phase 3 로드맵).
- **프레임리스 창:** 네이티브 타이틀바 없음(`decorations: false`), 타이틀바 띠도 없다.
  창 컨트롤(최소화·최대화·닫기)은 네비 레일 맨 위 구석에 끼워 넣고, 레일 자체를 드래그
  핸들(`<nav data-tauri-drag-region>`)로 써서 작업·AI 패널이 창 최상단까지 올라온다
  (macOS 오버레이 방식). 데스크탑 전용(웹 빌드는 미렌더). 구현:
  `components/WindowControls.tsx` + `Sidebar` 레일, 권한은 `capabilities/default.json`의
  `core:window:allow-*`.

### 네비게이션 모델

| 패널     | 키         | 단축키   | 성격             | 데이터 소스            |
| -------- | ---------- | -------- | ---------------- | ---------------------- |
| 오늘     | `today`    | ⌘/Ctrl+0 | 대시보드(브리핑) | 기존 리소스 fan-out    |
| 할일     | `todo`     | ⌘/Ctrl+1 | 그리드(CRUD)     | `todo`                 |
| 문서     | `doc`      | ⌘/Ctrl+2 | 에디터/뷰        | —                      |
| 메일     | `mail`     | ⌘/Ctrl+3 | 그리드 + 행 액션 | `mail`                 |
| 일정     | `calendar` | ⌘/Ctrl+4 | 그리드           | `calendar`             |
| 위키     | `wiki`     | ⌘/Ctrl+5 | 검색+편집 에디터 | `memory.*` (쿼리 구동) |
| 검색     | `search`   | ⌘/Ctrl+6 | 쿼리 구동        | `search.all`           |
| 연락처   | `people`   | ⌘/Ctrl+7 | 읽기 그리드      | `people`               |
| 크론     | `crons`    | ⌘/Ctrl+8 | 그리드 + 행 액션 | `crons`                |
| 작업피드 | `workfeed` | ⌘/Ctrl+9 | 그리드 + 액션 칩 | `workfeed`             |

- 활성 탭은 **떠오른다(lift)** — 흰 배경 + `--shadow-pill` + 아이콘이 액센트색으로.
  Zen 브라우저 탭 은유.
- 레일 라벨은 한국어 고정. 라틴 마이크로 라벨(`DENEB AI`)만 트래킹 대문자.

## 3. 디자인 토큰

전 토큰은 `src/styles.css`의 `:root`에 정의되고 `src/theme.ts`가 인라인용으로 미러한다.

### 3.1 색상

| 토큰             | 값                      | 용도                                                             |
| ---------------- | ----------------------- | ---------------------------------------------------------------- |
| `--grad`         | radial(웜) + linear(쿨) | 윈도우 배경 (고정). 흰 패널이 떠 보이도록 캔버스를 약간 가라앉힘 |
| `--panel`        | `#ffffff`               | 떠 있는 패널·필드 표면                                           |
| `--panel-sunken` | `#f7f7f9`               | 가라앉은 표면 (행 hover·row-btn hover)                           |
| `--ink`          | `#1f2128`               | 본문 1차 텍스트                                                  |
| `--ink-2`        | `#3e434c`               | 본문 2차 (그리드 셀·AI 응답)                                     |
| `--muted`        | `#6f747e`               | 보조 텍스트                                                      |
| `--muted-2`      | `#9a9ea7`               | 라벨·메타·placeholder                                            |
| `--faint`        | `#bcbfc6`               | 가장 흐린 힌트·비활성 점                                         |
| `--line`         | `#ececef`               | 헤어라인 (그리드 행·섹션 구분)                                   |
| `--line-2`       | `#e2e3e6`               | 컨트롤 보더 (필드·버튼)                                          |
| `--accent`       | `#c17a5b`               | **유일한 포인트 컬러** (점토)                                    |
| `--accent-deep`  | `#a85f43`               | 액센트 hover/press                                               |
| `--accent-soft`  | `rgba(193,122,91,.13)`  | 활성 배경·칩·선택 영역                                           |
| `--online`       | `#6b9b80`               | 연결됨 상태점 (차분한 세이지 그린)                               |
| `--due`          | `#a85038`               | 위험·지연·삭제·오류 (`color.danger`)                             |

**의미 색 규칙:** 색은 의미로만 쓴다. 장식용 색 금지.

- 점토 `--accent` = 강조·활성·1차 액션
- 그린 `--online` = 연결됨/정상
- 적벽돌 `--due` = 위험·삭제·오류·마감 임박
- 회색조(`muted`/`faint`) = 비활성·보조

### 3.2 형태(Shape)

| 토큰             | 값      | 용도                           |
| ---------------- | ------- | ------------------------------ |
| `--radius-panel` | `14px`  | 떠 있는 패널                   |
| `--radius-ctl`   | `9px`   | 필드·버튼                      |
| `--radius-pill`  | `8px`   | 네비 탭·작은 표면              |
| (칩)             | `999px` | 작업피드 액션 칩 (완전 둥글게) |

### 3.3 입체감(Elevation)

부유감은 **3겹 그림자**로 만든다 — 얇은 접촉(contact) + 중간 리프트(lift) + 깊고
넓은 앰비언트(ambient). 앰비언트 층이 "캔버스 위로 높이 떠 있다"는 인상의 대부분을
담당한다. 약한 단일 그림자보다 부유감이 또렷하다.

| 토큰             | 값(요지)                                                                      | 용도                     |
| ---------------- | ----------------------------------------------------------------------------- | ------------------------ |
| `--shadow-panel` | 접촉 `0 1px 2px .05` + 리프트 `…-10px .20` + 앰비언트 `0 30px 64px -22px .34` | 떠 있는 패널 (강한 부유) |
| `--shadow-pill`  | 접촉 `0 1px 2px .10` + 리프트 `0 8px 18px -5px .24`                           | 떠오른 활성 네비 탭      |

그림자는 **부유(floating) 표현 전용**이다. 강조·장식용 그림자 추가 금지. 부유감은
이 그림자 + 가라앉은 `--grad` 캔버스 + 패널 사이 `--gap` 여백, 셋의 합작이다.

### 3.4 레이아웃 토큰

| 토큰       | 값      | 용도            |
| ---------- | ------- | --------------- |
| `--rail-w` | `100px` | 네비 레일 폭    |
| `--ai-w`   | `352px` | AI 패널 폭      |
| `--gap`    | `12px`  | 셸 패딩·열 간격 |

### 3.5 간격 척도(권장)

엄격한 토큰은 아니지만 코드 전반의 사실상 척도. 신규 작업도 이 값들을 따른다.

```
2 · 3 · 7 · 8 · 9 · 12 · 13 · 18 · 20 · 22  (px)
```

패널 내부 18px, 작업영역 `20/22`, 컨트롤 패딩 `8×11`(필드)·`8×13`(버튼), 그리드 셀
`9×10`. 새 간격은 가까운 기존 값으로 스냅한다.

## 5. 컴포넌트 라이브러리

모든 컴포넌트는 `styles.css` 클래스(상태·구조) 또는 `theme.ts` 토큰(인라인)으로
만든다. 아래는 정식 카탈로그.

### 5.1 패널 — `.panel`

떠 있는 둥근 흰 표면. 흰 배경 + `--radius-panel`(14) + `--shadow-panel`. 작업 영역·
AI 패널·팝오버의 공통 그릇.

### 5.2 네비 탭 — `.nav-item` / `.nav-item.active`

| 상태   | 표현                                                                      |
| ------ | ------------------------------------------------------------------------- |
| 기본   | 투명 배경, `--muted` 텍스트, 아이콘 `--muted-2`                           |
| hover  | 반투명 흰 배경(`rgba(255,255,255,.6)`), 텍스트 `--ink-2`                  |
| active | 흰 배경 + `--shadow-pill`로 **떠오름**, 텍스트 `--ink`, 아이콘 `--accent` |

진입 시 26ms 간격 staggered `fade-up`. 전이 150ms.

### 5.3 폼 컨트롤 — `.field` / `.btn` / `.btn-accent`

| 컴포넌트      | 기본                                     | 상호작용                                               |
| ------------- | ---------------------------------------- | ------------------------------------------------------ |
| `.field`      | 흰 배경, `--line-2` 보더, radius 9, 13px | focus: 보더 액센트 + `0 0 0 3px var(--accent-soft)` 링 |
| `.btn`        | 흰 배경, `--line-2` 보더, 13px           | hover: `#f5f5f7` / active: `scale(.98)`                |
| `.btn-accent` | 액센트 배경, 흰 글자 (1차 액션)          | hover: `--accent-deep`                                 |
| `:disabled`   | opacity 0.5, 커서 기본                   | —                                                      |

1차 액션 1개당 `.btn-accent` 하나. 나머지는 `.btn` 또는 `.row-btn`.

### 5.4 데이터 그리드 — `.dgrid` + `Grid` / `GridNotice` / `RowBtn`

밀도 높은 표. 선언적 컬럼(`Column<T>`)을 `Grid`에 넘기면 `.dgrid` 크롬이 입혀진다.

- **헤더:** 11px tracked, `--muted-2`, 하단 `--line` 1px.
- **셀:** `9×10` 패딩, 13px, `--ink-2`, 행마다 `--line` 헤어라인.
- **행 hover:** `#faf9f8`(아주 옅은 웜).
- **행 액션 `RowBtn`:** 텍스트 버튼(읽음/보관/삭제/실행). hover 시 가라앉은 배경.
  `danger` prop이면 `--due`색.
- **`GridNotice`:** 그리드가 행을 못 그릴 때 안내만 렌더. **검사 순서 고정** — 연결 안 됨
  → 오류 → 로딩 → 빈 상태 → (그 외) 자식. 실패한 RPC를 "데이터 없음"으로 오인하지 않게.

### 5.5 칩 — `.chip`

작업피드 액션 선택 등. `--accent-soft` 배경, `--accent-deep` 글자, 완전 둥근 알약.

### 5.6 상태점 — `.live-dot` / `.live-dot.pulse`

6px 원. 연결됨 = `--online` + `pulse`(2.4s ease 호흡), 끊김 = `--faint` 정지.
레일 하단·AI 패널 헤더에 위치.

### 5.7 능동 알림 — `ProactivePanel`

데네브가 events SSE로 미는 넛지. 도착 전엔 **아무것도 렌더하지 않아** 조용할 땐
비켜선다. 각 넛지는 좌측에 2px 액센트 룰 + 제목/본문 + 닫기(×).

### 5.8 섹션 헤더 — `.sec-head` / `.micro`

`.sec-head`: baseline 정렬 + 하단 헤어라인. `.micro`: 트래킹 대문자 라틴 라벨
(`DENEB AI`, `게이트웨이`).

### 5.9 아이콘 — `Icon`

의존성 없는 라인 아이콘 세트(`Icon.tsx`). Tabler 스타일 아웃라인, 24 viewBox,
`currentColor`, stroke 1.85, round cap/join. 부모 색 상속, `size` prop으로 크기.

```
today todo doc mail calendar wiki search people crons workfeed   (패널)
send bell arrow-right close                                       (크롬)
```

새 아이콘은 같은 규격(24 viewBox·1.85 stroke·outline)으로 `PATHS`에 추가하고
`IconName`에 등록. 채움(fill) 아이콘·외부 아이콘 라이브러리 금지.

## 6. 상태 · 피드백

### 6.1 연결 상태

| 상태   | 레일 점           | AI 헤더 점        | 안내                                        |
| ------ | ----------------- | ----------------- | ------------------------------------------- |
| 연결됨 | `--online` + 펄스 | `--online` + 펄스 | 호스트명 표시                               |
| 끊김   | `--faint` 정지    | `--faint` 정지    | "연결" / "게이트웨이에 연결하면 표시됩니다" |
| 오류   | —                 | —                 | `--due`색 오류 메시지(팝오버·그리드)        |

게이트웨이 폼은 레일 하단 상태점의 **팝오버**에 숨긴다 — 실제 호스트에선 키체인에서
자동 연결되므로 거의 안 쓴다.

### 6.2 그리드 상태 매트릭스 (`GridNotice`)

| 조건       | 표현                                                    |
| ---------- | ------------------------------------------------------- |
| 연결 안 됨 | `muted` "게이트웨이에 연결하면 표시됩니다 (좌측 하단)." |
| 오류       | `--due` "불러오기 실패: {메시지}"                       |
| 로딩       | `muted` "불러오는 중…"                                  |
| 빈 결과    | `muted` 패널별 빈 문구("메일이 없습니다." 등)           |
| 정상       | 그리드 렌더                                             |

**오류를 빈 상태보다 먼저** 검사한다. 모든 오류 텍스트는 `errText()`로 정규화.

### 6.3 AI 스트림 피드백

- 응답 전: "메시지를 보내면 응답이 스트리밍됩니다."
- 사고 중: `thinking` 텍스트를 이탤릭 `--muted`로 한 줄.
- 응답 중: delta가 `--ink-2` `<pre>`에 줄바꿈 유지로 누적, 입력은 비활성("응답 중…").
- 컨텍스트 안내: "현재 뷰(X)를 읽는 중 · 도구로 바꾸면 즉시 반영됩니다" (`--faint`).

## 7. 모션 · 애니메이션

절제된 모션만. 모든 키프레임/전이는 `styles.css`에 있고, `prefers-reduced-motion`에서
0.001ms로 사실상 끈다.

| 모션         | 정의                                                | 용도                              |
| ------------ | --------------------------------------------------- | --------------------------------- |
| `fadeUp`     | 6px 상승 + 페이드, `cubic-bezier(.2,.7,.2,1)` 0.45s | 진입(네비 탭 26ms·섹션 60ms 간격) |
| `pulse`      | opacity 1↔0.4, 2.4s ease 무한                       | 연결 상태점 호흡                  |
| 컨트롤 전이  | background/color/box-shadow 0.12–0.15s ease         | hover·focus                       |
| `btn:active` | `scale(.98)` 0.05s                                  | 눌림 피드백                       |

**금지:** 그라디언트 애니메이션, 큰 슬라이드/바운스, 패럴랙스, 끄덕임. 모션은 진입·
상태·미세 피드백까지만.

## 8. 상호작용 패턴

### 8.1 키보드

- **⌘/Ctrl + 0–9** → 패널 전환. 레지스트리의 `shortcut`에서 파생(`Workstation.tsx`).
- **Enter** → AI 입력 전송, 위키/검색 쿼리 실행.
- **포커스 링:** `:focus-visible` 시 2px 액센트 아웃라인. 키보드 사용자 항상 보임.

### 8.2 AI ↔ 패널 컨텍스트 (핵심 패턴)

활성 패널은 자기 내용을 **텍스트로 직렬화**해 `useRegisterPane(resource, aiText)`로
공유한다. AI 패널은 그 텍스트를 읽어 컨텍스트로 삼는다.

```
패널 (그리드/에디터)  ──serializeList()/수동 직렬화──▶  useRegisterPane
                                                            │
                                                   workspaceContext
                                                            ▼
                                                   AIPanel이 읽어 전송
```

- **리스트형:** `serializeList("라벨", rows, row => "- …")` — 카운트 헤더 + 행당 한 줄.
- **에디터형:** 본문 텍스트 자체(`[위키 path]\n{content}`).
- **대시보드형:** 섹션별 카운트 헤더 + 캡 라인(`TodayPane`).
- 화면에 보이는 정보 == AI가 읽는 텍스트. 새 패널은 반드시 이걸 한다.

### 8.3 행 액션 & 쓰기

그리드 행 우측에 `RowBtn`으로 인라인 액션(읽음·보관·삭제·실행). 삭제류는 `danger`.
액션은 RPC 호출 후 그리드를 refetch. AI의 도구 호출이 데이터를 바꾸면 활성 그리드도
자동 갱신(`useChat`).

### 8.4 능동 알림

데네브가 미는 넛지는 AI 패널 상단에 쌓인다. 사용자가 닫을 때까지 유지, 조용하면 0개로
숨음. 작업 흐름을 가로막지 않는 비차단(non-blocking) 표면.

### 8.5 패널 유형별 UX

| 유형        | 예시                | 패턴                                                        |
| ----------- | ------------------- | ----------------------------------------------------------- |
| 읽기 그리드 | People·Calendar     | `useList` → `serializeList` → `GridNotice`+`Grid`           |
| 액션 그리드 | Mail·Crons·Workfeed | 위 + 행 `RowBtn`/`chip` + refetch                           |
| 쿼리 구동   | Search              | 입력 → `callRpc` 직접 → 로컬 상태 (data provider 미사용)    |
| 검색+편집   | Wiki                | 좌 검색 리스트 + 우 textarea 에디터, 저장 버튼·상태 텍스트  |
| 대시보드    | Today               | 기존 리소스 fan-out → 섹션별 브리핑 카드(클릭 시 해당 패널) |

## 9. 접근성

| 항목          | 처리                                                              |
| ------------- | ----------------------------------------------------------------- |
| 포커스 가시성 | `:focus-visible` 2px 액센트 아웃라인 + offset                     |
| 모션 민감     | `prefers-reduced-motion: reduce` → 전 애니메이션 0.001ms          |
| 아이콘 버튼   | `aria-label`(전송·닫기) + `title` 툴팁. 장식 아이콘 `aria-hidden` |
| 색 의존 금지  | 미읽음은 색(●)+굵기 둘 다, 상태는 점+텍스트                       |
| 최소 글자     | 11px 하한, 본문 정보 12px 이상                                    |
| 대비          | `--ink`(#1f2128) on `--panel`(#fff) ≈ 14:1. 본문은 `--ink-2` 사용 |

주의: `--muted-2`(#9a9ea7) on 흰색은 대비가 낮으므로 **라벨·메타 전용**, 본문 텍스트엔
쓰지 않는다.

## 10. 새 화면 만들 때 체크리스트

새 패널·컴포넌트가 시스템에 맞는지 출시 전 점검:

- [ ] 색은 `var(--…)` 토큰/`theme.ts`만. 생 hex·임의 색 없음.
- [ ] 점토 액센트 외 새 포인트 컬러 도입 안 함.
- [ ] 패널은 `.panel`, 컨트롤은 `.field`/`.btn`, 그리드는 `.dgrid`/`Grid` 재사용.
- [ ] `useRegisterPane`로 내용을 텍스트 직렬화해 AI에 공급.
- [ ] 연결 안 됨/오류/로딩/빈 상태를 `GridNotice`(또는 동등 분기)로 처리.
- [ ] 1차 액션 1개만 `.btn-accent`. 위험 액션은 `--due`/`danger`.
- [ ] 폰트 ≥12px(라벨 11px), 제목은 `h2`/`h3`.
- [ ] 아이콘은 `Icon` 세트 규격(24 viewBox·1.85 stroke·outline).
- [ ] 모션은 `fade-up`/기존 전이 재사용. 새 큰 모션 없음.
- [ ] 아이콘 버튼에 `aria-label`. 포커스 가능.
- [ ] 패널 추가면 `panes/index.ts`에 한 줄 등록(탭·단축키 자동 파생).

> 레시피(리소스 그리드 추가·쿼리 구동 패널 추가)는 [`CLAUDE.md`](../CLAUDE.md)의
> "Recipe" 절을, 아키텍처·데이터 흐름은 [`DESIGN.md`](DESIGN.md)를 참조.
