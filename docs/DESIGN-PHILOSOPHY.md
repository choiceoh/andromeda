# Andromeda 디자인 철학

> 우리가 디자인에 대해 **무엇을 믿는가**, 그리고 그 믿음으로 **어떻게 판단하는가**.
> 이 문서는 픽셀이나 토큰이 아니라 _태도_ 를 다룬다. 시스템 위에 있는 헌법이다.

문서 지도:

- [`DESIGN.md`](DESIGN.md) — 제품의 정체성·아키텍처 (**무엇을** 만드나)
- [`UI-UX.md`](UI-UX.md) — 디자인 **시스템** (**어떻게** 보이고 작동하나: 토큰·컴포넌트·패턴)
- **이 문서** — 디자인 **신념** (**왜** 그렇게 믿는가 + 설계할 때의 판단법)

`UI-UX.md` §1이 신념의 _요약_ 이라면, 이 문서는 그 _확장판_ 이다. 규칙이 헷갈릴 때
규칙을 더 외우지 말고, 여기로 돌아와 _왜_ 를 다시 읽어라.

---

## 한 문장

> **종일 켜두어도 피로하지 않은, 밀도 높지만 고요한 — 그라디언트 위에 떠 있는
> 따뜻한 작업 사령탑.**

이 문장과 충돌하는 결정은 틀린 결정이다. 화려함, 차가움, 산만함, 답답함 — 전부 이
문장에 진다.

---

## 정체성이 디자인을 결정한다

Andromeda는 문서 도구가 아니라 **종일 켜두는 사령탑**이다. 메일·일정·할일·위키·AI가
한 화면에 모인다. 이 사실이 모든 디자인을 강제한다:

- **종일** 본다 → 피로하면 안 된다 → 차분함은 사치가 아니라 요구사항.
- **많이** 본다 → 정보 밀도는 타협 불가 → 그러나 밀도가 소음이 되면 안 된다.
- **AI와 함께** 본다 → 데네브는 옆에서 같은 데이터를 본다 → 화면은 사람과 AI가
  공유하는 하나의 진실이어야 한다.

그래서 우리의 어려운 과제는 한 문장으로 요약된다: **"많이 보여주되, 시끄럽지 않게."**
아래 신념들은 전부 이 과제에 대한 답이다.

---

## 여섯 신념

### 1. 밀도 속의 고요

정보가 많다는 건 양보할 수 없다. 그렇다면 고요는 _빼기_ 로 만든다, 더하기가 아니라.
**밀도는 콘텐츠가 만들고, 고요는 크롬(chrome)이 만든다.** 크롬이 조용할수록 데이터가
또렷해진다. 그래서 헤어라인(1px), 균등폭 숫자, 작은 행간, 단 하나의 강조색. 표가
빽빽해도 화면이 비명을 지르지 않는 이유다.

### 2. 가두지 않고 띄운다

박스에 가두면 답답하고, 띄우면 숨 쉰다. 패널은 하드 보더로 칸막이된 게 아니라 부드러운
그라디언트 위에 **그림자로 떠 있다.** 이 부유감이 밀도 높은 앱을 가볍게 만든다.

> _이번 세션의 증거:_ 부유감이 약하다고 그림자를 3겹(접촉·리프트·앰비언트)으로
> 강화했고, 네이티브 타이틀바를 없애 패널이 창 맨 위까지 떠오르게 했다. 둘 다 "띄운다"는
> 신념의 직접 실행이다.

### 3. 따뜻함은 인간성, 액센트는 하나

업무 도구는 차가운 회색 대시보드가 되기 쉽다. 우리는 거부한다. 따뜻한 **클레이 액센트
하나** + 웜-쿨 그라디언트가 "내 비서의 작업대"라는 인간적 온도를 준다. 액센트가 하나인
이유: **두 번째 포인트 컬러는 강조가 아니라 소음이다.** 색은 _의미_ 일 때만 쓴다 —
정상은 초록, 위험은 적벽돌, 강조는 클레이. 장식으로 쓰는 색은 없다.

### 4. AI는 같은 화면을 읽는 동료

데네브는 위에서 지켜보는 감시자가 아니라 **옆에서 같은 데이터를 보는 동료**다. 그래서
**"화면에 보이는 것 == AI가 읽는 텍스트."** 모든 패널은 자기 내용을 텍스트로 직렬화해
AI에게 넘긴다(`useRegisterPane`). 픽셀이 아니라 텍스트. 이건 기술 결정이자 철학이다 —
사람과 AI가 _하나의 진실_ 을 공유한다. 새 화면을 만들 때 "AI가 이걸 텍스트로 읽을 수
있나?"를 못 물으면, 그 화면은 아직 안 끝났다.

### 5. 절제는 장식의 반대가 아니라 기능이다

그라디언트를 장식으로, 그림자를 강조로, 애니메이션을 즐거움으로 쓰지 않는다. **모든
시각 요소는 _의미_ 로 자격을 얻는다** — 의미 색, 기능적 그림자(=부유), 진입 모션(=맥락
전환). 자격 없는 요소는 뺀다. 망설여지면 뺀다. 절제가 만든 여백이 곧 고요다.

### 6. 일관성이 곧 완성도

**디자인은 가장 덜 다듬어진 요소만큼만 좋다.** 회색 스크롤바 하나, 네이티브 타이틀바
하나가 전체를 깬다. 그래서 우리는 가장자리까지 쫓는다 — 스크롤바, 창 크롬, 빈 상태,
포커스 링, 비활성 점. 토큰을 _법_ 으로 삼는 이유도 이것: 하드코딩 색 하나가 시스템에
균열을 낸다.

> _이번 세션의 증거:_ "디자인이 깨지잖아"라는 한마디에 스크롤바를 warm-Zen에 맞춰
> 다시 짰다(평소 투명 → hover 시 따뜻하게). 가장자리 하나가 전체의 완성도다.

## 취향 — 집는 것과 버리는 것

신념은 추상적이지만 취향은 구체적이다. 의심스러우면 이 목록으로 판정한다.

| 집는 것 (reach for)                       | 버리는 것 (reject)                  |
| ----------------------------------------- | ----------------------------------- |
| 웜 뉴트럴, 부드러운 웜-쿨 그라디언트      | 장식용 그라디언트, 메시·노이즈      |
| 헤어라인(1px), 가라앉은 표면              | 사방 하드 보더, 강조용 그림자       |
| 단 하나의 클레이 액센트                   | 복수 포인트 컬러, 무지개 색상       |
| 부유하는 둥근 패널, 그림자로 뜬 표면      | 칸막이 박스, 꽉 찬 프레임           |
| 조용한 진입 페이드·상태 펄스              | 튀는 바운스·슬라이드·패럴랙스       |
| 균등폭 숫자(tnum), Pretendard             | 들쭉날쭉 숫자, 장식 서체            |
| hover에 등장하는 컨트롤(스크롤바·창 버튼) | 항상 떠 있는 시끄러운 크롬          |
| 의미 기반 색(정상·진행·위험)              | 차가운 엔터프라이즈 그레이 대시보드 |
| ≥12px, 또렷한 대비                        | 12px 미만, 흐릿한 본문              |

## 긴장과 해소

좋은 디자인은 가치가 충돌하는 지점에서 결정된다. 우리가 충돌을 푸는 방향:

| 긴장              | 해소 방향                                                         |
| ----------------- | ----------------------------------------------------------------- |
| 밀도 ↔ 고요       | **고요 쪽으로.** 더하지 말고 빼서 푼다.                           |
| 부유 ↔ 가독       | 그림자가 _띄우고_, 콘텐츠는 불투명 흰 위에 또렷하게.              |
| AI 존재감 ↔ 집중  | AI는 오버레이가 아니라 _옆 패널_. 텍스트를 읽지 시선을 뺏지 않음. |
| 개성 ↔ 프로페셔널 | 절제된 뉴트럴 시스템 위, 딱 _하나_ 의 따뜻한 액센트.              |
| 자동화 ↔ 수작업   | 토큰·레지스트리로 일관성을 _기계적으로_ 강제, 손맛은 액센트에만.  |

## 설계할 때 스스로 묻는 일곱 질문

새 화면·컴포넌트·색·모션을 더하기 전에:

1. 이 요소는 _의미_ 가 있나, 장식인가? — 장식이면 뺀다.
2. 종일 봐도 피로하지 않나?
3. 토큰만 썼나, 생(raw) 색을 박았나?
4. 액센트를 _두 번째_ 로 늘리고 있진 않나?
5. AI가 이 내용을 텍스트로 읽을 수 있나?
6. 가장자리(스크롤바·빈 상태·포커스·연결 안 됨)까지 일관적인가?
7. 망설여지나? → **뺀다.**

이 질문들에 자신 있게 답하지 못하면, 그 디자인은 아직 끝나지 않았다.

## 명시적 거부 (non-goals)

신념만큼이나 _안 하는 것_ 이 정체성을 만든다:

- **다크 모드 없음** — 라이트 전용 warm-Zen이 정체성. 두 모드는 두 디자인이다.
- **장식 효과 없음** — 강조용 그림자, 장식 그라디언트, 글로우, 네온.
- **복수 액센트 없음** — 포인트 컬러는 영원히 하나.
- **차가운 대시보드 미감 거부** — 우리는 ERP가 아니라 _개인 비서의 작업대_ 다.
- **튀는 모션 없음** — 모션은 진입·상태·미세 피드백까지. 즐거움을 위한 애니메이션은 없다.
